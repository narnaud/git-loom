# Spec 018: Add (Stage Files)

## Overview

`git loom add` stages files into the git index using short IDs, plain
filenames, or the `zz` shorthand for staging everything. It is the loom
equivalent of `git add`, enhanced with short-ID resolution.

## Why Add?

Within a loom integration branch, working tree changes are identified by
short IDs in `loom status` output. Using `git add` requires the user to
type full paths. `loom add` lets users stage files using the same compact
short IDs they see in the status display.

## CLI

```bash
git-loom add <files...>
```

**Arguments:**

- `<files...>`: One or more files to stage. Required — at least one argument
  must be provided. Accepts:
  - **Short IDs**: compact identifiers from `loom status` output (e.g. `a3`, `0f`)
  - **Plain filenames**: relative paths (e.g. `src/main.rs`)
  - **`zz`**: stages all changes (like `git add -A`)

**Staging behavior:**

- Short IDs and filenames: stages only those files
- `zz`: stages all unstaged changes (equivalent to `git add -A`)
- `zz` mixed with other args: `zz` wins (stages everything)
- No args: rejected at CLI level (clap enforces `required = true`)

## Plain Staging

### What Happens

1. Open the repository.
2. If any argument is `"zz"`, stage all changes and return.
3. Otherwise, resolve each argument via the short-ID resolver:
   - If it matches a file short ID → use the resolved file path
   - If it matches a plain filename in the working tree → use it directly
   - If it matches nothing → error with a descriptive message
4. Stage all resolved files.
5. Print a confirmation: `"Staged N file(s)"`

### Error Cases

| Condition | Message |
|-----------|---------|
| No arguments | Blocked by clap (`required = true`) — shows usage |
| Short ID not found | Standard resolve_arg error: `"Could not resolve '<id>' as a file"` |
| File does not exist | Standard resolve_arg error |

### Examples

```bash
# Stage a single file by short ID
$ loom add a3

# Stage multiple files
$ loom add a3 0f src/lib.rs

# Stage everything
$ loom add zz
```
